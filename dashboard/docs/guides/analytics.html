Sentiment Analysis & Reputation Monitoring

Real-time sentiment analysis, reputation monitoring, alerting, and reporting for X/Twitter ‚Äî no API fees.

Overview

XActions Analytics provides a complete sentiment intelligence layer:

  • Sentiment Analysis ‚Äî Rule-based (offline, instant) or LLM-powered (OpenRouter) analysis of any text
  • Reputation Monitoring ‚Äî Continuous polling of mentions/keywords with rolling sentiment tracking
  • Alert System ‚Äî Threshold, volume spike, and anomaly detection with webhook + Socket.IO delivery
  • Reports ‚Äî Comprehensive Markdown/JSON reports with timelines, keyword frequency, and distribution

Available via: API, CLI, MCP tools (for AI agents), and Dashboard.

API Reference

Base URL: /api/analytics

POST /sentiment

Analyze sentiment of text or batch of texts.

Single text:

// Request
{ "text": "I love this! üî•", "mode": "rules" }

// Response
{
  "score": 0.65,
  "label": "positive",
  "confidence": 0.45,
  "keywords": ["love", "üî•"]
}

Batch mode (up to 100 texts):

// Request
{ "texts": ["great!", "terrible", "it's ok"], "mode": "rules" }

// Response
{
  "results": [
    { "text": "great!", "score": 0.6, "label": "positive", "confidence": 0.5, "keywords": ["great"] },
    { "text": "terrible", "score": -0.6, "label": "negative", "confidence": 0.5, "keywords": ["terrible"] },
    { "text": "it's ok", "score": 0, "label": "neutral", "confidence": 0, "keywords": [] }
  ],
  "count": 3
}

Parameters:

Field Type Default Description
text string ‚Äî Single text to analyze
texts string[] ‚Äî Array of texts for batch analysis (max 100)
mode string "rules" "rules" (offline) or "llm" (OpenRouter)

Score range: -1.0 (very negative) to 1.0 (very positive)
Labels: positive (>0.05), neutral (-0.05 to 0.05), negative (<-0.05)

Sentiment Engine Details

Rule-Based Mode (rules)

Fast, offline analysis using an AFINN-style lexicon:

  • ~150 scored words with scores from -5 (worst) to +5 (best)
  • Negation handling ‚Äî "not", "never", "don't" etc. flip and weaken scores by 25%
  • Intensifiers ‚Äî "very" (1.5x), "extremely" (2x), "slightly" (0.5x)
  • Emoji scoring ‚Äî 20+ emojis mapped to sentiment (üòÄ=3, üò°=-3, üî•=2)
  • Normalization ‚Äî Scores normalized to -1.0 to 1.0 range

Best for: High-volume analysis, offline use, cost-sensitive workloads.

LLM Mode (llm)

Uses OpenRouter to call an LLM for nuanced analysis:

  • Default model: meta-llama/llama-3.1-8b-instruct:free
  • Handles sarcasm, irony, crypto/finance slang, internet culture
  • Structured JSON output with graceful fallback to rule-based on parse failure
  • Requires OPENROUTER_API_KEY environment variable

Best for: Complex/nuanced text, sarcasm detection, specialized domains.

Dashboard

Access at /analytics ‚Äî a dark-themed dashboard with four tabs:

  1. Analyze ‚Äî Quick text analysis with batch support, sentiment badges, and keyword tags
  2. Monitors ‚Äî Start/stop monitors, view status table with rolling averages and trends
  3. Timeline ‚Äî Chart.js line+bar chart showing sentiment over time with mentions overlay, word cloud, mention feed
  4. Alerts ‚Äî Color-coded alert history with severity badges and timestamps

Features:

  • Real-time alert notifications via Socket.IO
  • Ctrl+Enter keyboard shortcut for analysis
  • Responsive layout (sidebar collapses on mobile)
  • Accessible (ARIA roles, skip navigation, focus indicators)
